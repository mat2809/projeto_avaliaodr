# Requisitos do Sistema de Controle de Loja

## 1. Requisitos Funcionais

RF01 - O sistema deve permitir o cadastro de clientes com nome e e-mail.  
RF02 - O sistema deve permitir o cadastro, edição e exclusão de produtos.  
RF03 - O sistema deve permitir o cadastro de usuários para login no sistema.  
RF04 - O sistema deve permitir o login e autenticação de usuários.  
RF05 - O sistema deve registrar vendas realizadas com múltiplos itens.  
RF06 - O sistema deve calcular automaticamente o total da venda com base nos itens.  
RF07 - O sistema deve permitir a visualização da lista de clientes cadastrados.  
RF08 - O sistema deve verificar o estoque disponível antes de registrar uma venda.  
RF09 - O sistema deve atualizar o estoque dos produtos após a venda.  
RF10 - O sistema deve exibir mensagens de sucesso ou erro ao usuário.

## 2. Requisitos Não Funcionais

RNF01 - O sistema deve utilizar a linguagem Python com a biblioteca Tkinter para a interface gráfica.  
RNF02 - O sistema deve seguir a arquitetura MVC (Model-View-Controller).  
RNF03 - O sistema deve utilizar o banco de dados MySQL para armazenamento de dados.  
RNF04 - Os dados de conexão com o banco devem ser configurados via arquivo `.env`.  
RNF05 - O sistema deve ter no mínimo 3 janelas diferentes na interface.  
RNF06 - O sistema deve ser versionado com GitHub, com commits a cada 20 minutos.  
RNF07 - A interface gráfica deve ser simples, funcional e intuitiva para o usuário final.  
RNF08 - O código deve estar organizado em pastas: `models/`, `views/`, `controllers/`, `database/`.  
RNF09 - Toda documentação deve estar presente na pasta `/docs` do projeto.  
RNF10 - O projeto deve conter README.md completo e bem escrito.

## 3. Restrições

RES01 - O sistema deve ser desenvolvido individualmente.  
RES02 - O projeto deve conter no mínimo 5 tabelas relacionadas no banco de dados.  
RES03 - O repositório GitHub deve se chamar `projeto_avaliador` e ser público.  
RES04 - A entrega deve ser realizada em até 4 horas, com apresentação ao professor.
